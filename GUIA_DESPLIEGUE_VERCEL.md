# 🚀 Guía de Despliegue en Vercel

Esta guía te ayudará a desplegar el proyecto OsorIA Landing Page en Vercel.

## 📋 Requisitos Previos

1. Cuenta en [Vercel](https://vercel.com) (gratuita)
2. Cuenta en [Supabase](https://supabase.com) (gratuita)
3. Repositorio Git (GitHub, GitLab o Bitbucket) conectado

## 🔧 Paso 1: Configurar Variables de Entorno en Vercel

### Variables REQUERIDAS

Estas variables son **obligatorias** para que la aplicación funcione correctamente:

```env
NEXT_PUBLIC_SUPABASE_URL=https://tu-proyecto.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu_clave_anonima_aqui
SUPABASE_SERVICE_ROLE_KEY=tu_clave_de_servicio_aqui
SUPABASE_URL=https://tu-proyecto.supabase.co
```

### Variables OPCIONALES (para el Chatbot con IA)

Estas variables son **opcionales** pero recomendadas para mejorar el chatbot:

```env
# Opción 1: DeepSeek (recomendado, más económico)
DEEPSEEK_API_KEY=tu_clave_deepseek_aqui

# Opción 2: OpenAI (alternativa)
OPENAI_API_KEY=tu_clave_openai_aqui

# Opción 3: XAI/Grok (para chat avanzado)
XAI_API_KEY=tu_clave_xai_aqui
```

**Nota:** Si no configuras ninguna API key de IA, el chatbot funcionará con el sistema de FAQ basado en palabras clave.

### Cómo Configurar las Variables en Vercel

1. Ve a tu proyecto en [Vercel Dashboard](https://vercel.com/dashboard)
2. Selecciona tu proyecto
3. Ve a **Settings** → **Environment Variables**
4. Agrega cada variable de entorno:
   - **Name**: Nombre de la variable (ej: `NEXT_PUBLIC_SUPABASE_URL`)
   - **Value**: Valor de la variable
   - **Environment**: Selecciona Production, Preview y Development (o al menos Production)
5. Haz clic en **Save**
6. Repite para cada variable

## 🔗 Paso 2: Obtener Credenciales de Supabase

### Si ya tienes un proyecto de Supabase:

1. Ve a tu proyecto en [Supabase Dashboard](https://app.supabase.com)
2. Ve a **Settings** (⚙️) → **API**
3. Copia estos valores:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL` y `SUPABASE_URL`
   - **anon public** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role** → `SUPABASE_SERVICE_ROLE_KEY` ⚠️ (mantén esto secreto)

### Si necesitas crear un proyecto nuevo:

1. Ve a [supabase.com](https://supabase.com) y crea una cuenta
2. Crea un nuevo proyecto
3. Espera a que se complete la configuración (2-3 minutos)
4. Sigue los pasos anteriores para obtener las credenciales

## 🗄️ Paso 3: Configurar la Base de Datos en Supabase

### Crear las Tablas Necesarias

1. Ve a tu proyecto en Supabase Dashboard
2. Ve a **SQL Editor** (en el menú lateral)
3. Ejecuta los siguientes scripts SQL en orden:

#### 3.1. Tabla de Usuarios

Ejecuta el contenido de `migrations/users.sql`:

```sql
CREATE TABLE IF NOT EXISTS users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

#### 3.2. Tabla de Conversaciones del Chat

Ejecuta el contenido de `migrations/chat_conversations.sql`:

```sql
CREATE TABLE IF NOT EXISTS chat_conversations (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_chat_conversations_session_id ON chat_conversations(session_id);
CREATE INDEX IF NOT EXISTS idx_chat_conversations_created_at ON chat_conversations(created_at);
```

#### 3.3. Tabla de Envíos de Contacto (opcional)

Ejecuta el contenido de `migrations/contact_submissions.sql` si lo tienes.

## 🚀 Paso 4: Desplegar en Vercel

### Opción A: Desde el Dashboard de Vercel

1. Ve a [vercel.com/new](https://vercel.com/new)
2. Conecta tu repositorio (GitHub, GitLab o Bitbucket)
3. Selecciona el repositorio `OsorIALanginPage`
4. Vercel detectará automáticamente que es un proyecto Next.js
5. Verifica la configuración:
   - **Framework Preset**: Next.js
   - **Root Directory**: `./` (raíz del proyecto)
   - **Build Command**: `pnpm build` (o `npm run build`)
   - **Output Directory**: `.next` (automático)
6. **No hagas clic en Deploy todavía**
7. Ve a **Environment Variables** y agrega todas las variables (ver Paso 1)
8. Haz clic en **Deploy**

### Opción B: Desde la Línea de Comandos

```bash
# Instalar Vercel CLI
npm i -g vercel

# Iniciar sesión en Vercel
vercel login

# Desplegar
vercel

# Para producción
vercel --prod
```

## ✅ Paso 5: Verificar el Despliegue

Después del despliegue, verifica:

1. **Página principal**: Debe cargar correctamente
2. **Chatbot**: Debe funcionar (abre el botón flotante)
3. **Formulario de contacto**: Debe enviar mensajes
4. **Autenticación**: Login/Registro debe funcionar
5. **Base de datos**: Las conversaciones deben guardarse en Supabase

### Comandos de Verificación

```bash
# Verificar que el build funciona localmente
pnpm build

# Iniciar en modo producción local
pnpm start
```

## 🔍 Troubleshooting (Solución de Problemas)

### Error: "Variables de entorno no configuradas"

- Verifica que todas las variables estén configuradas en Vercel
- Asegúrate de que estén disponibles en el ambiente **Production**
- Reinicia el despliegue después de agregar variables

### Error: "La tabla no existe en Supabase"

- Verifica que ejecutaste los scripts SQL en Supabase
- Revisa que la tabla `chat_conversations` existe en tu proyecto de Supabase
- Verifica los permisos de RLS (Row Level Security) en Supabase

### El chatbot no responde

- Verifica las variables de entorno de Supabase
- Revisa los logs de Vercel en **Deployments** → selecciona el deployment → **Functions** → `/api/chat-simple`
- Si usas IA, verifica que las API keys sean correctas

### Error de build en Vercel

- Verifica que `next.config.mjs` esté configurado correctamente
- Revisa los logs del build en Vercel
- Asegúrate de que todas las dependencias estén en `package.json`
