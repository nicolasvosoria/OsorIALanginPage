# 🚀 Guía Rápida de Instalación - Windows

## Pasos Rápidos para Desplegar Localmente

### 1️⃣ Verificar Node.js
Abre PowerShell o CMD y verifica que tienes Node.js instalado:
```powershell
node --version
```
Si no lo tienes, descárgalo de [nodejs.org](https://nodejs.org/) (versión 18 o superior)

### 2️⃣ Instalar pnpm
```powershell
npm install -g pnpm
```

### 3️⃣ Instalar dependencias
En la carpeta del proyecto, ejecuta:
```powershell
pnpm install
```

### 4️⃣ Configurar Supabase

#### a) Crear cuenta y proyecto en Supabase
1. Ve a [supabase.com](https://supabase.com)
2. Crea una cuenta (gratis)
3. Crea un nuevo proyecto
4. Espera a que se complete la configuración (2-3 minutos)

#### b) Obtener las credenciales
1. En tu proyecto de Supabase, ve a **Settings** (⚙️) > **API**
2. Copia estos valores:
   - **Project URL** → será tu `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** → será tu `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role** → será tu `SUPABASE_SERVICE_ROLE_KEY` ⚠️ (mantén esto secreto)

#### c) Crear archivo .env.local
En la raíz del proyecto, crea un archivo llamado `.env.local` con este contenido:

```env
NEXT_PUBLIC_SUPABASE_URL=https://tu-proyecto.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu_clave_anonima_aqui
SUPABASE_SERVICE_ROLE_KEY=tu_clave_de_servicio_aqui
SUPABASE_URL=https://tu-proyecto.supabase.co
```

Reemplaza los valores con los que copiaste de Supabase.

### 5️⃣ Configurar la Base de Datos

1. En Supabase, ve a **SQL Editor** (en el menú lateral)
2. Ejecuta cada uno de estos archivos SQL (copia y pega el contenido):

**Archivo 1: `migrations/users.sql`**
```sql
CREATE TABLE IF NOT EXISTS users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, now()) NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, now()) NOT NULL,
  last_login TIMESTAMP WITH TIME ZONE
);

CREATE INDEX IF NOT EXISTS idx_users_email ON users(email);
```

**Archivo 2: `migrations/contact_messages.sql`**
```sql
CREATE TABLE IF NOT EXISTS contact_messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT,
  idea TEXT NOT NULL,
  status TEXT DEFAULT 'nuevo' CHECK (status IN ('nuevo', 'en_proceso', 'completado')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_contact_messages_created_at ON contact_messages(created_at);
CREATE INDEX IF NOT EXISTS idx_contact_messages_status ON contact_messages(status);
CREATE INDEX IF NOT EXISTS idx_contact_messages_email ON contact_messages(email);
```

**Archivo 3: `migrations/chat_conversations.sql`**
```sql
CREATE TABLE IF NOT EXISTS chat_conversations (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_chat_conversations_session_id ON chat_conversations(session_id);
CREATE INDEX IF NOT EXISTS idx_chat_conversations_created_at ON chat_conversations(created_at);
```

3. Haz clic en **RUN** después de cada SQL

### 6️⃣ Ejecutar el Proyecto

```powershell
pnpm dev
```

Abre tu navegador en: **http://localhost:3000**

- ✅ Sin errores en la terminal de PowerShell

## 🔧 Solución de Problemas Comunes

### Error: "Cannot find module"
```powershell
# Elimina node_modules y reinstala
Remove-Item -Recurse -Force node_modules
pnpm install
```

### Error: "Falta la clave de servicio de Supabase"
- Verifica que `.env.local` existe en la raíz del proyecto
- Verifica que las claves están correctas (sin espacios extra)
- Reinicia el servidor: `Ctrl+C` y luego `pnpm dev` de nuevo

### Error: "relation does not exist"
- Asegúrate de haber ejecutado los 3 archivos SQL en Supabase
- Verifica que las tablas aparecen en **Table Editor** de Supabase

### Puerto 3000 ya está en uso
```powershell
# Usa otro puerto
pnpm dev -- -p 3001
```
