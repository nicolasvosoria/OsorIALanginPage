# Modern SaaS Landing Page

## Variables de Entorno Requeridas

Para que esta aplicación funcione correctamente, necesitas configurar las siguientes variables de entorno:

### Supabase (Obligatorias)

\`\`\`env
# La URL de Supabase ya está configurada en el código
# SUPABASE_URL=https://hiyotpejjfpwzhhghcar.supabase.co

# Clave de servicio de Supabase (para operaciones administrativas)
SUPABASE_SERVICE_ROLE_KEY=tu_clave_de_servicio_aquí

# Clave anónima de Supabase (para operaciones del cliente)
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu_clave_anónima_aquí
\`\`\`

## Cómo obtener las claves de API de Supabase

1. Inicia sesión en tu [Dashboard de Supabase](https://app.supabase.io)
2. Selecciona tu proyecto
3. Ve a Configuración > API
4. Bajo "Project API keys", encontrarás:
   - `anon public`: Esta es tu `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `service_role secret`: Esta es tu `SUPABASE_SERVICE_ROLE_KEY` (¡mantenla segura!)

## Configuración de la Base de Datos

La aplicación utiliza Supabase como base de datos. Asegúrate de tener las siguientes tablas creadas:

### Tabla `contact_messages`

\`\`\`sql
CREATE TABLE IF NOT EXISTS contact_messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT,
  idea TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  status TEXT DEFAULT 'pending'
);

CREATE INDEX IF NOT EXISTS idx_contact_messages_email ON contact_messages(email);
CREATE INDEX IF NOT EXISTS idx_contact_messages_created_at ON contact_messages(created_at);
CREATE INDEX IF NOT EXISTS idx_contact_messages_status ON contact_messages(status);
\`\`\`

### Tabla `users`

\`\`\`sql
CREATE TABLE IF NOT EXISTS users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, now()) NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, now()) NOT NULL,
  last_login TIMESTAMP WITH TIME ZONE
);

CREATE INDEX IF NOT EXISTS idx_users_email ON users(email);
\`\`\`

## Desarrollo Local

1. Clona este repositorio
2. Crea un archivo `.env.local` con las variables de entorno necesarias (basado en `.env.example`)
3. Instala las dependencias: `npm install`
4. Inicia el servidor de desarrollo: `npm run dev`
